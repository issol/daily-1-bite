"구글에 검색한다"는 말이 곧 바뀔지도 모릅니다. 더 정확히는, 검색하는 **방식**이요.

3월 26일, Google이 Search Live를 미국 한정에서 **200개국 이상**으로 확대했습니다. 동시에 이를 구동하는 새 모델 **Gemini 3.1 Flash Live**를 공개했고요. 이제 한국어를 포함한 90개 이상의 언어로, 스마트폰 카메라를 들이대며 AI와 대화할 수 있습니다.

타이핑 대신 말하고, 스크린샷 대신 카메라를 비추는 검색. 실제로 써보면 어떤 경험인지, 그리고 개발자에게는 어떤 의미인지 정리해봤습니다.

## TL;DR

- **Search Live**: Google 검색에서 음성 + 카메라로 실시간 대화하는 AI 검색 기능
- 3월 26일부터 **200개국, 90개+ 언어** 지원 (한국 포함)
- **Gemini 3.1 Flash Live**: 새 오디오 모델, 배경 소음 필터링, 감정 인식, 대화 길이 2배 향상
- 모든 AI 음성 출력에 **SynthID 워터마크** 자동 삽입
- 개발자용 **Gemini Live API** (Google AI Studio)에서 프리뷰 제공
- Flash Lite 가격: 입력 $0.25/1M 토큰, 출력 $1.50/1M 토큰

## Search Live가 뭔가요?

![AI 음성 검색](https://images.unsplash.com/photo-1763610452177-9eb2b1ae8853?w=1080&q=80)
_Photo by [Muhammad Numan](https://unsplash.com/@numaninsari?utm_source=daily-1-bite&utm_medium=referral) on [Unsplash](https://unsplash.com/?utm_source=daily-1-bite&utm_medium=referral) | 검색의 인터페이스가 텍스트에서 음성과 카메라로 전환되고 있습니다_

Search Live는 Google 앱에서 **음성과 카메라**를 통해 검색하는 기능입니다. 기존 검색과의 핵심 차이를 정리하면요.

**기존 검색:** 키워드 입력 → 링크 목록 → 클릭해서 확인

**Search Live:** 말하거나 카메라 비추기 → AI가 음성으로 답변 → 후속 질문으로 대화 이어가기

예를 들어, IKEA 선반을 조립하다가 막히면 설명서에 카메라를 들이대고 "이 부품 어디에 끼우는 거야?"라고 물으면 됩니다. AI가 카메라 화면을 실시간으로 분석하고, 음성으로 답변해줍니다.

### 실제 사용 흐름

1. Google 앱을 열고 검색바 아래 **Live 아이콘** 탭
2. 음성으로 질문하거나 카메라를 켜서 대상을 비춤
3. AI가 음성으로 답변 + 화면에 관련 웹 링크 표시
4. 후속 질문으로 대화를 이어감
5. 다른 앱으로 전환해도 **백그라운드에서 대화 유지**

마지막 포인트가 꽤 중요합니다. 요리하면서 레시피를 물어보고, 타이머 앱을 켜도 Search Live 대화가 끊기지 않거든요.

## Gemini 3.1 Flash Live: 무엇이 달라졌나

Search Live의 품질을 결정하는 건 결국 뒤에서 돌아가는 모델입니다. Gemini 3.1 Flash Live는 Google이 "역대 최고 품질 오디오 모델"이라고 부르는 녀석인데요, 주요 개선점을 보겠습니다.

### 1. 대화 지속 능력 2배

이전 모델 대비 **대화 스레드를 2배 더 길게** 추적할 수 있습니다. "아까 그 식당 얘기했잖아, 거기 예약 어떻게 해?"처럼 이전 맥락을 참조하는 질문에도 잘 대응한다는 뜻이에요.

### 2. 배경 소음 필터링

카페에서, 지하철에서, 공사 현장 옆에서도 사용자의 음성을 정확히 분리합니다. 실외 환경에서의 사용성이 크게 올라갔어요.

### 3. 감정 인식 및 톤 조절

사용자의 **피치(음높이)와 속도**를 분석해서 감정 상태를 추론합니다. 답답해하는 것 같으면 답변을 짧고 직접적으로, 궁금해하는 것 같으면 더 상세하게 조절해요.

이건 단순한 편의 기능이 아닙니다. 음성 인터페이스에서 "답변이 너무 길다/짧다"는 가장 흔한 불만인데, 이걸 톤으로 자동 조절한다는 건 UX 관점에서 꽤 큰 진전이에요.

### 4. 네이티브 다국어 지원

설정을 바꾸지 않아도 **사용자가 말하는 언어를 자동 감지**합니다. 한국어로 시작했다가 영어 문서를 보여주면 영어로 전환하는 것도 가능해요. 90개 이상 언어를 지원합니다.

## 개발자가 주목할 3가지

![AI 검색 기술](https://images.unsplash.com/photo-1763610452303-b7a10b3d4462?w=1080&q=80)
_Photo by [Muhammad Numan](https://unsplash.com/@numaninsari?utm_source=daily-1-bite&utm_medium=referral) on [Unsplash](https://unsplash.com/?utm_source=daily-1-bite&utm_medium=referral) | 개발자에게 Search Live는 새로운 검색 인터페이스이자 API 기회입니다_

### 1. Gemini Live API 접근

Gemini 3.1 Flash Live는 **Google AI Studio**에서 개발자 프리뷰로 사용할 수 있습니다. 자사 앱에 실시간 음성 + 비전 AI를 붙이고 싶다면 주목할 만한 API예요.

**Gemini 3.1 Flash 모델 라인업 가격:**

| 모델 | 입력 (1M 토큰) | 출력 (1M 토큰) | 특징 |
|------|---------------|---------------|------|
| **3.1 Flash Lite** | $0.25 | $1.50 | 속도 최적화, GPQA 86.9% |
| **3.1 Pro** | $2.00 | $12.00 | ARC-AGI-2 77.1%, 범용 최강 |

Flash Lite는 이전 2.5 Flash 대비 **첫 토큰 응답 시간 2.5배 향상**, 출력 속도 45% 향상을 달성했습니다. 초당 232.5 토큰을 생성하는데, 실시간 음성 응답에 필요한 속도를 충분히 확보한 셈이죠.

### 2. SynthID 워터마크와 AI 콘텐츠 규제

모든 Search Live 음성 출력에는 **SynthID 워터마크**가 자동으로 삽입됩니다. 사람 귀에는 들리지 않지만, 기계적으로 감지할 수 있는 방식이에요.

이건 [EU AI Act의 AI 콘텐츠 표시 의무](https://daily1bite.com/blog/ai-tutorial/google-ai-mode-ai)와 직결되는 기술입니다. AI가 생성한 음성인지 아닌지를 기술적으로 판별할 수 있게 되면, 딥페이크 방지와 규제 준수가 한결 수월해지거든요.

개발자 입장에서도 중요합니다. 자사 서비스에 AI 음성을 도입한다면, 워터마크 삽입은 곧 필수 요건이 될 가능성이 높아요.

### 3. 검색 패러다임 전환의 영향

Search Live의 글로벌 확대는 "검색 = 텍스트 입력"이라는 20년 된 공식을 깨뜨립니다. 이전에 [Google AI Mode 분석](https://daily1bite.com/blog/ai-tutorial/google-ai-mode-ai)에서 다뤘듯이 검색 인터페이스 자체가 바뀌고 있어요.

**SEO/콘텐츠 전략에 미치는 영향:**

- 음성 검색은 키워드가 아니라 **자연어 문장**으로 질문함 → 롱테일 키워드 중요성 증가
- 카메라 검색은 **이미지/비주얼 콘텐츠**의 검색 노출 비중을 높임
- AI가 직접 답변하므로 **구조화된 데이터**(Schema.org) 마크업이 더 중요해짐

[블로그 SEO 최적화 글](https://daily1bite.com/blog/seo/blog-seo-3-weeks)에서도 다뤘지만, 이 변화에 맞춰 콘텐츠 전략도 업데이트가 필요합니다.

## 경쟁 구도: 누가 음성 AI 검색을 이끄나?

Google만 이런 걸 하는 게 아닙니다.

| 서비스 | 음성 검색 | 카메라 검색 | 다국어 | 특징 |
|--------|----------|-----------|--------|------|
| **Google Search Live** | O | O | 90개+ | Gemini 3.1 Flash Live, SynthID |
| **Apple Siri + Intelligence** | O | O (예정) | 40개+ | 온디바이스 처리, 프라이버시 중심 |
| **OpenAI ChatGPT Voice** | O | O | 50개+ | GPT-5.4 기반, 추론 능력 강점 |
| **Samsung Galaxy AI** | O | O | 16개 | 온디바이스 + 클라우드 하이브리드 |

Google의 강점은 **검색 데이터와의 통합**입니다. 단순히 AI가 답변하는 게 아니라, 웹 검색 결과를 실시간으로 참조하면서 답변하고, 관련 링크까지 제공하거든요. [Apple Intelligence](https://daily1bite.com/blog/ai-tools/apple-intelligence-2026-llm-siri-gemini)가 프라이버시를 강점으로 내세운다면, Google은 검색 인프라를 무기로 삼는 셈이에요.

## 냉정한 평가

기대되는 점만큼 우려도 있습니다.

**프라이버시 우려는 여전합니다.** 카메라와 마이크를 통해 수집되는 데이터의 양이 텍스트 검색과는 비교할 수 없이 많아요. Google은 "음성 데이터를 서버에 저장하지 않는다"고 하지만, 실시간으로 서버에 전송되는 건 맞습니다.

**네트워크 의존성이 높습니다.** Search Live는 온디바이스가 아니라 클라우드 기반이라, 네트워크 상태가 나쁘면 경험이 크게 떨어집니다. 지하철이나 엘리베이터에서는 쓰기 어려울 수 있어요.

**검색 품질은 검증이 필요합니다.** 90개 언어를 지원한다고 해도, 한국어처럼 주요 언어가 아닌 경우 품질 차이가 있을 수 있습니다. 실제 한국어 성능은 직접 써봐야 알 수 있어요.

**생태계 종속 심화.** Search Live가 편할수록 Google 검색 의존도가 높아지고, 이는 Google의 시장 지배력을 더욱 강화합니다. 경쟁 검색 엔진이나 독립 콘텐츠 플랫폼 입장에서는 더 불리해지는 구조예요.
