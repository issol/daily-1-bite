4월 첫째 주에 Claude Code가 네 번 업데이트됐습니다. v2.1.89부터 v2.1.92까지, 릴리스 노트를 읽으면서 "이건 꽤 쓸만하겠다"는 기능들이 여럿 눈에 띄었어요.

특히 MCP 결과 크기 500K 확장, `/powerup` 인터랙티브 학습, 서브에이전트 관련 버그 수정들이 인상적이었습니다. 매일 Claude Code를 쓰는 입장에서 하나씩 뜯어봤습니다.

## TL;DR

- **`/powerup`**: 애니메이션 데모가 포함된 인터랙티브 레슨 (v2.1.90)
- **MCP 500K**: `_meta["anthropic/maxResultSizeChars"]`로 최대 50만 자 결과 저장 (v2.1.91)
- **`defer` 권한 결정**: PreToolUse 훅에서 헤드리스 일시정지/재개 지원 (v2.1.89)
- **`/cost` 상세 내역**: 모델별·캐시 히트별 비용 분석 (v2.1.92)
- **서브에이전트 안정성**: tmux 창 종료 후 서브에이전트 재생성 버그 수정 (v2.1.92)
- **퍼포먼스**: Write 도구 diff 계산 60% 빨라짐, SSE 트랜스포트 선형 시간으로 개선

## 하이라이트 1: `/powerup` — 인터랙티브 학습 레슨

![코드 화면](https://images.unsplash.com/photo-1568716353609-12ddc5c67f04?w=1080&q=80)
_Photo by [Patrick Martin](https://unsplash.com/@patrickmmartin?utm_source=daily-1-bite&utm_medium=referral) on [Unsplash](https://unsplash.com/?utm_source=daily-1-bite&utm_medium=referral) | Claude Code의 새로운 /powerup으로 인터랙티브하게 기능을 익힐 수 있습니다_

v2.1.90에서 추가된 `/powerup`은 단순한 도움말이 아닙니다. **애니메이션 데모**와 함께 Claude Code의 기능을 단계별로 배울 수 있는 인터랙티브 레슨이에요.

Claude Code를 처음 쓰는 팀원한테 "써봐, 어렵지 않아"라고 해도 진입 장벽이 꽤 있었는데, 이제 `/powerup`을 켜놓고 따라 하면 됩니다. 온보딩 시간을 눈에 띄게 줄여줄 것 같아요.

아직 레슨 목록 전체가 공개되진 않았지만, Anthropic이 이 방향으로 계속 콘텐츠를 추가할 것으로 보입니다.

## 하이라이트 2: MCP 결과 500K 자 저장

이전까지 MCP 툴 결과의 기본 크기 제한은 꽤 작았습니다. 큰 파일을 MCP 도구로 읽어오면 결과가 잘리거나 별도의 처리가 필요했어요.

v2.1.91부터는 `_meta["anthropic/maxResultSizeChars"]`를 사용해서 **최대 500,000 자까지** 결과를 저장할 수 있습니다.

```json
{
  "_meta": {
    "anthropic/maxResultSizeChars": 500000
  }
}
```

[MCP 프로토콜 활용법](https://daily1bite.com/ko/blog/ai-tutorial/mcpmodel-context-protocol-ai-2026)에서 소개했듯이, MCP는 Claude Code의 확장성 핵심입니다. 이번 업데이트로 대규모 코드베이스나 긴 문서를 MCP 도구로 처리할 때 훨씬 유연해졌습니다.

실제로 큰 로그 파일을 분석하거나, 여러 파일을 한 번에 불러오는 MCP 서버를 만들 때 이 제한이 걸림돌이었는데, 이제는 실용적으로 쓸 수 있겠습니다.

## 하이라이트 3: `defer` 권한 결정 — 헤드리스 일시정지/재개

![코드 컨텍스트 메뉴](https://images.unsplash.com/photo-1763568258533-d0597f86ce62?w=1080&q=80)
_Photo by [Daniil Komov](https://unsplash.com/@dkomow?utm_source=daily-1-bite&utm_medium=referral) on [Unsplash](https://unsplash.com/?utm_source=daily-1-bite&utm_medium=referral) | defer 권한 결정으로 헤드리스 환경에서도 세밀한 제어가 가능해졌습니다_

v2.1.89에서 추가된 `defer` 권한 결정은 **자동화 파이프라인**을 구성하는 개발자에게 특히 유용합니다.

기존에는 PreToolUse 훅에서 권한을 허가하거나 거부하는 두 가지 선택만 있었습니다. 이제 `"defer"`를 반환하면 Claude Code가 **일시정지**하고, 외부 시스템이 준비되면 재개할 수 있습니다.

```json
{
  "decision": "defer",
  "resumeAfter": "external-check-complete"
}
```

CI/CD 파이프라인에서 특정 도구 사용 전 외부 승인을 받거나, 멀티 에이전트 워크플로우에서 체크포인트를 만들 때 활용할 수 있습니다. `PermissionDenied` 훅과 재시도 지원도 같이 추가됐어요.

## 하이라이트 4: `/cost` 상세 내역 — 모델별·캐시 히트별 분석

v2.1.92에서 `/cost` 명령어가 크게 업그레이드됐습니다. 이전에는 세션 전체 비용만 보여줬다면, 이제는:

- **모델별 분류**: 어떤 모델이 얼마나 사용됐는지
- **캐시 히트 분류**: 캐시 히트로 절약된 비용이 얼마인지

[Claude Code 제작기](https://daily1bite.com/ko/blog/dev-life/claude-code-2-saas-utakoto)에서도 비용 관리의 중요성을 다뤘는데, 이제 훨씬 세밀하게 파악할 수 있습니다.

Pro 사용자는 **프롬프트 캐시 만료 힌트**도 추가로 볼 수 있습니다. 캐시가 언제 만료되는지 미리 알 수 있으니, 긴 세션에서 비용 최적화 타이밍을 잡기 더 쉬워집니다.

## 하이라이트 5: Named 서브에이전트 @멘션

v2.1.89에서 이름 있는 서브에이전트를 **`@멘션`으로 지목**할 수 있게 됐습니다.

이전에는 서브에이전트가 익명으로 실행됐는데, 이제 특정 서브에이전트에게 직접 지시를 내릴 수 있습니다. 멀티 에이전트 워크플로우에서 "어느 에이전트가 무슨 일을 하는지" 명확하게 구분할 수 있게 된 셈입니다.

## 성능 개선 & 버그 수정

### Write 도구 diff 계산 60% 빨라짐

v2.1.92에서 Write 도구의 diff 계산 알고리즘이 최적화됐습니다. 대용량 파일 작업이 많은 세션에서 체감 속도가 달라질 것입니다.

### SSE 트랜스포트 선형 시간으로 개선 (v2.1.90)

기존에 SSE(Server-Sent Events) 트랜스포트가 **O(n²)** 시간 복잡도로 동작해서, 긴 대화에서 성능이 급격히 저하되는 문제가 있었습니다. v2.1.90에서 **O(n)** 선형 시간으로 개선됐습니다. 장시간 세션을 자주 사용하는 분들에게 체감 차이가 클 거예요.

같은 v2.1.90에서 **긴 대화 트랜스크립트 저장**도 선형 시간으로 개선됐습니다.

### tmux 창 종료 후 서브에이전트 버그 수정

v2.1.92에서 tmux 창이 강제 종료된 후 서브에이전트 재생성이 실패하던 버그가 수정됐습니다. 멀티 에이전트를 무거운 환경에서 돌릴 때 간헐적으로 터지던 문제였는데 해결됐습니다.

### `--resume` 프롬프트 캐시 미스 수정

v2.1.69부터 이어져온 `--resume` 옵션에서 프롬프트 캐시 미스가 발생하던 회귀 버그가 v2.1.90에서 수정됐습니다.

### autocompact 스래시 루프 수정

v2.1.89에서 자동 컨팩트가 반복 루프에 빠지던 버그도 수정됐습니다. 긴 세션에서 컨텍스트가 압축될 때 비정상적으로 반복되던 동작이었어요.

### CJK/이모지 프롬프트 히스토리 드롭 수정

v2.1.89에서 한국어, 중국어, 일본어 등 CJK 문자와 이모지가 포함된 프롬프트 히스토리가 **조용히 삭제**되던 버그가 수정됐습니다. 한국어로 Claude Code를 쓰는 분들이라면 체감이 있었을 문제입니다.

## Codex CLI와 비교

[OpenAI Codex CLI 4월 업데이트](https://daily1bite.com/ko/blog/ai-tools/openai-codex-cli-april-2026-update)와 같은 시기에 나온 업데이트들을 비교해보면:

| 기능 | Claude Code (v2.1.89~92) | Codex CLI (v0.118) |
|------|--------------------------|-------------------|
| 학습 지원 | **`/powerup` 인터랙티브 레슨** | 도움말 텍스트 |
| MCP 확장성 | **500K 결과 크기** | MCP 네이티브 지원 |
| 권한 제어 | **`defer` 일시정지/재개** | 기본 권한 모델 |
| 비용 분석 | **모델별·캐시 히트별 상세** | 사용량 기본 표시 |
| 퍼포먼스 | SSE O(n²)→O(n) 개선 | **Spark 1,000+ TPS** |
| 서버 인증 | 기본 | **디바이스 코드 로그인** |

Claude Code는 MCP 확장성과 에이전트 제어 측면에서 강화됐고, Codex CLI는 속도와 Windows 호환성에 집중하고 있습니다.

## 제거된 기능

v2.1.92에서 `/tag`와 `/vim` 명령어가 제거됐습니다. 실제로 얼마나 쓰이는지 몰랐는데, 제거되는 것 보니 사용률이 낮았나봐요. vim 키바인딩을 쓰고 싶다면 외부 에디터 통합을 사용하는 게 공식 방향으로 보입니다.

## 냉정한 평가

이번 4월 첫째 주 업데이트들을 보면, Anthropic이 세 가지에 집중하고 있다는 느낌입니다:

1. **진입 장벽 낮추기** — `/powerup`으로 온보딩 개선
2. **확장성 강화** — MCP 500K, 서브에이전트 개선
3. **버그 사냥** — CJK 드롭, autocompact 루프, tmux 버그 등 실사용자 문제 해결

특히 CJK 프롬프트 히스토리 드롭 수정은 한국어 사용자들한테 체감이 있을 것 같습니다. 이런 작은 버그들이 쌓이면 실제로 도구 신뢰도에 영향을 주거든요.

`/vim` 제거는 아쉽지만, 프레임워크가 더 집중된 방향으로 가고 있다는 신호로 보입니다.

다음 업데이트에서는 무엇이 나올지 — 특히 에이전트 팀 기능이 더 성숙해지는 방향인지 지켜보겠습니다.
